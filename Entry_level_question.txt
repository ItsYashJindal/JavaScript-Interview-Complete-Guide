what is asynchronous/ synchronous ? 

ðŸ‘‰ In synchronous execution, JavaScript runs code line by line, and each operation must finish before the next one starts. If one task takes time, it blocks the main thread, and the remaining code has to wait until that task is completed. This means the program cannot move forward until the current operation finishes.

In asynchronous execution, JavaScript does not block the main thread for time-consuming tasks. When an asynchronous operation like a timer, API call, or Promise is triggered, JavaScript moves it to the background using Web APIs and continues executing the remaining code. Once the asynchronous task is completed, its result is handled later by the Event Loop.

what is event loop ? 

ðŸ‘‰ java script is single threaded but event loop allow us to handle async operation. event loop continuously checks, Is the call stack empty or not ,If yes, move pending tasks to the call stack. first of step all Synchronous code runs in the Call Stack, If the code is asynchronous, like setTimeout, Promise, or an API call, it is sent to Web APIs. Web APIs handle the task in the background. After the task is completed, it does not go back directly to the Call Stack. Instead, PromiseðŸ‘‰related tasks are sent to the Microtask Queue, and setTimeout or DOM events are sent to the Callback (Macrotask) Queue. The Event Loop keeps checking if the Call Stack is empty. When it is empty, the Event Loop first moves all tasks from the Microtask Queue to the Call Stack and executes them. Only after that, it takes tasks from the Callback Queue and moves them to the Call Stack one by one. This is how JavaScript manages asynchronous code even though it runs on a single thread.

what is async await ? 

ðŸ‘‰ Async/await is used to handle asynchronous operations efficiently in JavaScript. It is used inside functions, where the async keyword is used to create an async function, and an async function always returns a Promise. The await keyword pauses the execution of the async function until the Promise is resolved or rejected.

what is promises ?

ðŸ‘‰ A Promise in javascript is a way, which show an asynchronous task is completed or not. A Promise has three states: pending, fulfilled, and rejected. The pending state means the operation is still in progress, fulfilled means the operation has completed, and rejected means the operation has failed.

what is map/ reduce/ filter ? 

ðŸ‘‰ Map, filter, and reduce are JavaScript array methods used to work with array data in an easy and clean way. The map method is used when we want to perform an operation on every element of an array and return a new array with updated values. The filter method is used when we want only specific elements from an array based on a condition, for example getting all numbers greater than 4 from an array. The reduce method is used when we want to combine all elements of an array into a single value, such as calculating the total, sum, or any final result from the array data.
